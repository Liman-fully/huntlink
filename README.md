## ⚠️ 同步规则（简单！）

### 开发前（避免重复）

```
1. git pull（获取最新）
2. 查看 docs/TODAY_PROGRESS.md（看别人在做什么）
3. 查看 .task-board.md（看任务状态）
4. 确认无人做 → 领取任务
```

### 开发中

```
每完成一步 → git commit + git push
遇到报错 → 立即记录到 docs/TODAY_PROGRESS.md
```

### 离开前（必须完成！）

```
1. git pull（获取最新）
2. git add + git commit + git push（同步代码）
3. 更新 docs/TODAY_PROGRESS.md（进展 + 配置）
4. 更新 .task-board.md（任务状态）
5. 通知@协调者
```

### 查看进展（只需两个地方）

```bash
# 1. 今日进展（进展 + 配置）
cat docs/TODAY_PROGRESS.md

# 2. 任务状态
cat .task-board.md
```

**进度文档**：
- `docs/TODAY_PROGRESS.md` - **唯一进展文档（最重要！）**
- `.task-board.md` - **任务状态**

## 🚨 防重复开发机制

### 问题重现

```
都统开发 → 报错 → 没同步 → 离开
    ↓
新人加入 → 拉取代码 → 找不到进展 → 重新开发
    ↓
都统回来 → 发现重复 → 浪费 token
```

### 解决方案

**1. 强制同步**：
```
开发中：每 5 分钟 git pull
完成一步：立即 git push
遇到报错：立即同步 + 记录
离开前：必须 push + 记录进展
```

**2. 进展可视化**：
```
docs/TODAY_PROGRESS.md（实时进展）
.task-board.md（任务状态）
```

**3. 配置文档化**：
```
docs/config/[项目].md（配置说明）
```

**4. 新人检查清单**：
```
加入后先检查：
- [ ] docs/TODAY_PROGRESS.md（最新进展）
- [ ] .task-board.md（任务状态）
- [ ] docs/experiences/[ROLE]/accumulated.md（经验）
- [ ] git log --oneline -5（最新提交）
```
