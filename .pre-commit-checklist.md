# 提交前检查清单

每次提交代码前，必须完成以下检查：

## 1. 经验沉淀 ✅

### 必填项
- [ ] **本次修改解决了什么问题？**
- [ ] **关键代码变更点（最多3个）**
- [ ] **是否踩坑？踩了什么坑？如何解决的？**

### 可选项
- [ ] 技术决策理由
- [ ] 性能优化点
- [ ] 待办事项（TODO）

### 记录位置
- 日常记录：`.workbuddy/memory/YYYY-MM-DD.md`
- 长期经验：`.workbuddy/memory/MEMORY.md`
- 提交信息：git commit message

---

## 2. 代码质量 ✅

### 自动检查（CI）
- [ ] `npm run lint` 无错误
- [ ] `npm run test` 全部通过
- [ ] `npm run build` 构建成功

### 手动检查
- [ ] 无 console.log 调试代码
- [ ] 无硬编码密钥/密码
- [ ] 删除无用代码和注释

---

## 3. 问题总结 ✅

### 本机特有问题
- [ ] 记录本机特有的环境问题
- [ ] 记录依赖安装的特殊步骤
- [ ] 记录IDE/编辑器配置差异

### 通用问题
- [ ] 是否影响其他电脑？
- [ ] 是否需要更新 README？
- [ ] 是否需要数据库迁移？

---

## 4. 提交规范 ✅

### Commit Message 格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档变更
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动
- `perf`: 性能优化

### 示例
```
feat(resume): 实现简历解析模块

- 集成阿里SmartResume解析引擎
- 支持12种格式简历上传
- 自动提取关键信息并结构化存储

Closes #123
```

---

## 5. 版本迭代清理 ✅

### 每个版本发布前
- [ ] 清理 node_modules 重新安装
- [ ] 清理 dist/build 目录
- [ ] 清理测试缓存
- [ ] 清理日志文件
- [ ] 运行完整测试套件

### 清理命令
```bash
# 后端清理
cd backend && rm -rf node_modules dist && npm install && npm run build

# 前端清理
cd frontend-web && rm -rf node_modules dist && npm install && npm run build

# 测试缓存清理
npm run test -- --clearCache
```

---

## 快速检查脚本

```bash
# 运行所有检查
npm run lint && npm run test && npm run build

# 或者使用一键脚本
npm run pre-commit
```
