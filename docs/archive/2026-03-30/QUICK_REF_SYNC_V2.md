## 📌 汇报前必做（30 秒）

```bash
# 1. 运行核查脚本
./scripts/verify-task-status.sh [任务 ID]

# 2. 运行汇报前检查
./scripts/pre-report-check.sh
```

---

## ✅ 三源验证（CROSS-VERIFY）

| 信息源 | 检查内容 | 命令 |
|--------|---------|------|
| **任务看板** | 状态是否最新 | `cat .task-board.md` |
| **git 提交** | 是否有提交记录 | `git log --grep="任务 ID"` |
| **代码文件** | 文件是否存在 | `ls -la [路径]` |

**三者一致 → 可汇报**  
**三者不一致 → 先纠正再汇报**

---

## 🚨 禁止行为

- ❌ 仅看任务看板就汇报
- ❌ 未核查 git 提交就汇报
- ❌ 未检查代码文件就汇报
- ❌ 任务完成后不更新文档

---

## ✅ 正确做法

- ✅ 汇报前执行核查脚本
- ✅ 任务完成后立即更新文档
- ✅ 提交信息包含任务 ID
- ✅ 离开前强制同步所有文档
