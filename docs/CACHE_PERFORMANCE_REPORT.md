## TTL 策略优化

### 当前策略
| 场景 | TTL | 说明 |
|------|-----|------|
| 热门搜索 | 300s | 5 分钟 |
| 精确搜索 | 300s | 5 分钟 |
| 详情页 | 600s | 10 分钟 |
| 搜索建议 | 180s | 3 分钟 |
| 复杂查询 | 300s | 5 分钟 |

### 优化建议
| 场景 | 当前 TTL | 建议 TTL | 理由 |
|------|---------|---------|------|
| 热门搜索 | 300s | 900s | 高频访问，延长缓存时间 |
| 精确搜索 | 300s | 600s | 中频访问，适度延长 |
| 详情页 | 600s | 900s | 稳定数据，延长缓存 |
| 搜索建议 | 180s | 300s | 变化较快，保持较短 TTL |
| 复杂查询 | 300s | 600s | 计算密集，延长缓存 |

## 缓存键策略

### 当前格式
```
huntlink:cache:candidate:search:{paramsHash}
huntlink:cache:candidate:detail:{id}
huntlink:cache:candidate:suggestions:{keyword}
```

### 优化建议
```
huntlink:cache:v1:candidate:search:{paramsHash}:{version}
huntlink:cache:v1:candidate:detail:{id}:{updatedAt}
huntlink:cache:v1:candidate:suggestions:{keyword}:{timestamp}
```

**优势**:
- 支持版本控制
- 支持数据更新时自动失效
- 支持时间窗口失效

## 优化建议

### 短期优化（本周）
1. ✅ **提高热门搜索 TTL**: 从 5 分钟提升到 15 分钟
2. ✅ **添加缓存统计服务**: 实时监控命中率
3. ⏳ **添加预加载机制**: 对热门搜索词预加载
4. ⏳ **优化缓存键策略**: 使用更细粒度的缓存键

### 中期优化（本月）
1. ⏳ **实现多级缓存**: L1（内存）+ L2（Redis）
2. ⏳ **添加缓存预热**: 启动时预加载热门数据
3. ⏳ **实现缓存降级**: Redis 故障时自动降级到数据库
4. ⏳ **添加缓存分析**: 定期分析缓存使用模式

### 长期优化（下季度）
1. ⏳ **分布式缓存**: Redis Cluster 部署
2. ⏳ **智能缓存**: 基于 ML 预测缓存策略
3. ⏳ **缓存 CDN**: 静态数据 CDN 加速
