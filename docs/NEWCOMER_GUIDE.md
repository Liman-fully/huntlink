## 🚀 第三步：执行 5 分钟闭环（15 分钟）

### 时间分配

```
0:00 - 收到任务
  ↓
0:00-0:30   思考（第一性原理）
  ↓
0:30-1:00   技能检查（强制搜索）
  ↓
1:00-3:00   全速开发
  ↓
3:00-4:00   测试
  ↓
4:00-4:30   同步（更新任务看板）
  ↓
4:30-5:00   闭环（提交 + 通知）
```

### 详细流程

#### 1. 思考（30 秒）

问自己：
- 这个任务的目标是什么？
- 有没有现成的解决方案？
- 最优路径是什么？

#### 2. 技能检查（30 秒）

**强制搜索技能**：
```bash
npx skills search <关键词>
```

**检查已安装技能**：
```bash
npx skills list
```

**如果需要，安装技能**：
```bash
npx skills add <owner/repo@skill> -g -y
```

#### 3. 全速开发（2 分钟）

- 专注核心功能
- 避免过度设计
- 使用最佳实践

#### 4. 测试（1 分钟）

**测试清单**：
- [ ] 功能测试通过
- [ ] TypeScript 编译通过
- [ ] ESLint 检查通过

#### 5. 同步（30 秒）

**更新任务看板**：
```markdown
| TZS-003 | 跟进记录 | P0 | [设计稿](#我的主页设计) | ✅ 已完成 |
```

**添加完成信息**：
```markdown
**完成时间**: 2026-03-28 10:05
**实际耗时**: 30 分钟
**代码提交**: abc123
**测试**: ✅ 通过
```

#### 6. 闭环（30 秒）

**提交代码**：
```bash
git add .
git commit -m "feat: 完成跟进记录功能"
git push origin master
```

**通知左护法**：
```

```
